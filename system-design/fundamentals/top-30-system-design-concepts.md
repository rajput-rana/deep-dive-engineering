## When to Use

These concepts appear in virtually every large-scale system design:
- **Scalability concepts:** When designing for growth
- **Reliability patterns:** When uptime is critical
- **Data management:** When handling large datasets
- **Architecture patterns:** When structuring complex systems
- **Distributed systems:** When building multi-service systems
- **Networking:** When services need to communicate

## Interview Hints

When asked "What are the key concepts in system design?":
1. Group concepts by category (scalability, reliability, data, etc.)
2. Explain tradeoffs between related concepts
3. Give real-world examples for each
4. Show understanding of when to apply each concept
